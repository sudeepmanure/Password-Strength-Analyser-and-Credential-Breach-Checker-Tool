How k-Anonymity Works Security by design
1
Hash Locally
SHA-1 hash is computed entirely in your browser using the Web Crypto API. The plaintext password never leaves your device.
2
Send 5-Char Prefix
Only the first 5 hex characters of the hash are sent to api.pwnedpasswords.com — not the full hash, not the password.
3
Receive Hash Range
The API returns ~800 hash suffixes that match the prefix. This gives plausible deniability — the server never knows which hash you're checking.
4
Match Locally
Your browser compares the full hash suffix against the returned list. The result is determined entirely client-side — private by design.
Security Recommendations
  • 🔑Use a passphrase of 4+ random words (e.g., correct-horse-battery-staple) — these are long, memorable, and highly resistant to brute force.
  • 🛡️Never reuse passwords across multiple sites. A breach on one platform exposes all accounts using the same credential (credential stuffing).
  • 🗄️Use a password manager (Bitwarden, KeePass, 1Password) to generate and store unique 20+ character random passwords for every account.
  • 📱Enable Multi-Factor Authentication (MFA) wherever possible. Even a compromised password is useless without the second factor.
  • ⚠️Avoid dictionary words, keyboard patterns (qwerty, 12345), names, dates, and substitutions like p@ssw0rd — these are in every cracking wordlist (rockyou.txt, hashcat rules).
  • 🔄Change passwords only when breached — not on a fixed schedule. Forced rotation often leads to weaker, predictable passwords with increments.
